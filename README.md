Conforme os resultados apresentados acima, se compararmos as operações com ambos vetores de 100, 1000 e 10000 números entre si, podemos observar que nas colunas de 
`Tempo (ms)` e `Comparações`, os resultados do `Selection Sort` são aproximadamente 1/3 dos valores do `Bubble Sort`, enquanto para a coluna de `Movimentações`, os resultados do 
`Selection Sort` são consideravelmente menores do que os valores do `Bubble Sort`. Portanto, para esse contexto, com vetores invertidos de 100, 1000 e 10000 números, o `Selection Sort`
apresentou o melhor desempenho, uma vez que todos os valores para tempo de execução, número de comparações e número de movimentações são menores.
